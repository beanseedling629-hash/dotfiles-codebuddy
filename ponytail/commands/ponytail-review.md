# /ponytail-review — 审查当前改动的过度工程

对当前未提交的 diff（或用户指定的文件/代码段）做「过度工程」审查，给出可删清单。

## 执行步骤

1. 获取当前改动范围（git diff，或用户指定的文件）
2. 对每处改动套用判断阶梯，找出：
   - 不必要存在的代码（YAGNI）
   - 本可用标准库 / 原生功能 / 已有依赖替代的
   - 多余的抽象 / 包装 / 配置项 / 提前优化
   - 可压缩为更少行数的实现
3. 输出「删除/简化清单」，每条给出：位置 → 问题 → 建议的更简写法
4. **不动安全相关代码**（校验/错误处理/安全/可访问性），这些即使"看起来多"也保留
5. 仅给建议，不擅自改代码；用户确认后再改

## 输出格式

```
[位置] 问题 → 建议
例: utils/date.ts:12  自写日期格式化 → 用 Intl.DateTimeFormat（原生）
```
