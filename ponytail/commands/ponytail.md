# /ponytail — 切换防过度工程强度

调整 ponytail 规则（`~/.codebuddy/rules/ponytail.md`）的生效强度。

## 用法

`/ponytail [lite|full|ultra|off]`，不带参数时显示当前档位并列出可选项。

## 各档位含义

- `lite` — 仅在代码明显过度工程时才提醒，干扰最小
- `full` — 默认。每次写代码前走「判断阶梯」（需要存在吗→标准库→原生→已装依赖→一行→最小实现）
- `ultra` — 激进精简，优先质疑「这段代码是否需要存在」，最大化复用与最小实现
- `off` — 本次会话临时关闭 ponytail 约束

## 执行

1. 解析用户给的档位参数；非法参数则提示可选值
2. 在本次会话中按该档位调整对代码量/复杂度的克制程度
3. 回显：当前档位 + 一句话说明该档位行为
4. 始终保留安全红线（校验/错误处理/安全/可访问性），任何档位都不牺牲
