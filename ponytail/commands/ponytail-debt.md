# /ponytail-debt — 汇总延后的精简标记

把代码里用 `// ponytail:` 标记的「先延后、之后再精简」的点，整理成一份债务账本。

## 执行步骤

1. 全仓搜索标记：`ponytail:`（支持 `// ponytail:`、`# ponytail:`、`/* ponytail: */` 等注释形态）
2. 逐条收集：文件:行号 → 标记说明 → 周边上下文一句话
3. 汇总成账本，按可处理性 / 影响排序
4. 给出建议处理顺序；用户选定后可逐条转为实际精简（走 /ponytail-review 思路）

## 输出格式

```
| 位置 | 说明 | 建议 |
|------|------|------|
| api/user.ts:42 | 临时多包了一层 service | 内联，删包装 |
```

若全仓没有 `ponytail:` 标记，回复「当前无延后精简项」。
