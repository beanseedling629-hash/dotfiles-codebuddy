# /ponytail-audit — 审计整个仓库的过度工程

对整个项目（而非单次 diff）做过度工程审计，找出长期累积的冗余。

## 执行步骤

1. 浏览项目结构，识别重点区域（utils、abstractions、wrappers、config、未使用导出等）
2. 按判断阶梯逐类排查：
   - 只有单一实现却存在的抽象层 / 接口 / 工厂
   - 自己重造、实则标准库/原生/已有依赖已提供的功能
   - 死代码、未被引用的导出、从未走到的分支
   - 重复的第三方库（功能重叠）
   - 提前可配置化但从无第二种取值的参数
3. 汇总成分级清单（高/中/低收益），每条：位置 → 问题 → 简化方向 → 预估可减行数
4. 安全/校验/错误处理/可访问性相关一律不计入
5. 仅出报告，不擅自改；用户挑选后再逐项处理

## 输出

按「收益从高到低」排序的可精简清单 + 总览（预计可减代码量）。
